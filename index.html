Why do we should use Unit-tests?

Simplicity

Informativity

Parallel development

Possibility of reuse

What Unit-tests can not do?

Not all errors will be found

Will not show integration errors

Steps

Choose fragment

Call stimul method

Check results

Analysis of a separate module

Creating a case test

Testing the module

Unit test should:

Correspond to a specific module

Be automated

Be timely

Answer the main tasks

Have a good name

When unit test will not help:

testing complex and branched algorithms

Absence of clear results

Testing the code interacting with the system

Checking the entire application

Insufficient qualifications of the developer
